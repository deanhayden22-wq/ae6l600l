================================================================================
METHODOLOGY NOTES
================================================================================

Gap detection uses ADDRESS-BASED matching (not label name matching).
For each function, four hex formats are searched:
  - Raw stripped hex (e.g., "36A98")
  - 0x-prefixed (e.g., "0x36A98")
  - Full 8-digit (e.g., "00036A98")
  - 0x + 8-digit (e.g., "0x00036A98")

This avoids false positives caused by Ghidra labels not matching FUN_ references
in analysis files (e.g., CL/OL was 100% complete but appeared as 16 gaps under
name-based matching).
