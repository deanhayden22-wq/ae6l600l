ELECTRONIC THROTTLE BODY (ETB/DBW) — CONTROL ANALYSIS
AE5L600L (2013 Subaru WRX MT) — Renesas SH7058 (SH-2)
================================================================================

Code range: 0x050000 - 0x054851 (~18.4 KB)
Called from: task10_call52 (0x47C40) Dispatch Table A -- 13 entries
            task10_sub_0x049082 Dispatch Table B -- 28 entries
            Additional callers: 0x04A752, 0x04F19C, 0x0489C0

================================================================================
SECTION 1: OVERVIEW
================================================================================

The 0x050000-0x054851 range contains the Electronic Throttle Body (ETB)
drive-by-wire control system. This is the code that translates driver
pedal input into throttle plate position via the electronic throttle
actuator.

The code is organized as two dispatch tables called from within
task10 (20ms sensor task):

  Dispatch Table A @ 0x0483D4 (13 entries)
    Called from task10_call52 (0x47C40) — the same mega-function
    that also contains fuel pump control and CL/OL management.
    These appear to be the FAST path (sensor conditioning, position
    tracking, safety checks).

  Dispatch Table B @ 0x049760 (28 entries)
    Called from an internal sub-function at 0x049082 within the
    task10 call chain. These appear to be the FULL path (target
    computation, PID control, actuator output, limit enforcement).

Both tables dispatch sequentially — each entry is a sub-function
that processes one stage of the throttle control pipeline.

Slot [0] is the largest (314B) — reads ECT and throttle, evaluates
8 descriptors. This is the initial sensor conditioning that sets up
the DBW workspace for the rest of the pipeline.

Slot [5] (94B) processes accelerator pedal position — reads accel_state,
throttle, and throttle sub-bytes (FFFF65C6). This converts raw pedal
input into the requested throttle target.

Slot [11] (56B) processes clutch state (FFFF5BE3) alongside throttle —
likely the clutch-cancel/rev-match logic for manual transmission.

Slot [12] reads boost workspace (FFFF8B90, 8BD0) — boost-based
throttle limiting to protect the turbo/engine.

Key observations:

  - Slot [2] (626B) is the LARGEST function — reads ECT, atmospheric
    pressure, and throttle with 4 calibrations. This is likely the
    main throttle target computation.

  - Slot [4] (566B) reads engine state, throttle, and checks the
    fuel pump DTC flag (FFFF8E98). This is the safety interlock
    that limits throttle when engine faults are detected.

  - Slot [13] (394B, 14 calibrations) is the most calibration-heavy
    function. Reads engine state, ECT, throttle — likely the cold-start
    throttle opening compensation (idle air control via ETB).

  - Slot [15] has 20 calibrations — the throttle-to-torque mapping
    function that converts pedal input into a torque request.

  - Slot [25] (11 cals) and [27] (12 cals) access engine state +
    ECT + atmospheric pressure — altitude and temperature compensation
    for the throttle target.

  - Slot [26] reads clutch state — the manual transmission clutch
    intervention (throttle blip for downshifts or rev-hang reduction).

  - Slot [11] (444B, no cals/descs) is the pedal interpretation
    function — converts raw pedal into requested opening using the
    existing workspace values.

================================================================================
SECTION 7: INTEGRATION WITH OTHER SUBSYSTEMS
================================================================================

The ETB/DBW system interfaces with:

  1. Boost control (FFFF8B50-8BD0) — throttle limiting under boost
  2. Fuel pump (FFFF8E98) — safety throttle reduction on pump DTC
  3. CL/OL fueling (via task10_call52 shared parent)
  4. Ignition timing (FFFF8E7E timing state, read by slot B[23])
  5. Cruise control (likely via FFFF8A9C-8A9D workspace)
  6. Clutch/transmission (FFFF5BE3 clutch state, slots A[11], B[26])

The fact that this code runs WITHIN task10_call52 (the same function
as fuel pump and CL/OL management) explains why it was never identified
as a separate subsystem — it's deeply embedded in the sensor task's
sequential dispatch chain.


================================================================================
SECTION 8: SUMMARY
================================================================================

The 0x050000-0x054851 range is the Electronic Throttle Body (ETB)
drive-by-wire (DBW) control system comprising:

  - 13 fast-path functions (sensor conditioning, position tracking)
  - 28 full-path functions (target computation, PID, safety limits)
  - 26 descriptor tables for throttle mapping
  - 117 calibration parameters
  - ~790 bytes of RAM workspace (FFFF88E8-8BFF)

This fills the last major code gap between the fueling pipeline
(0x030000-0x03FFFF) and the boost control system (0x054852+).

The existing "Torque Management / DBW" analysis in
torque_management_analysis.txt covers the high-level torque request
path but not this low-level ETB actuator control code. This analysis
documents the actual throttle plate positioning implementation.
